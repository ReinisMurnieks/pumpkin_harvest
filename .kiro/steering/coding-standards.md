# Coding Standards

## Python (Flask Backend)
- Use Python 3.8+ features
- Follow PEP 8 style guide
- Keep routes simple and focused
- Use descriptive variable names
- Add comments for complex logic

## JavaScript (Game Logic)
- Use ES6+ features
- Prefer `const` and `let` over `var`
- Use descriptive function names
- Keep functions small and focused
- Add comments for game mechanics

## HTML/CSS
- Use semantic HTML5 elements
- Keep CSS organized by component
- Use flexbox/grid for layouts
- Maintain responsive design principles
- Use emojis for visual elements

## Canvas Rendering
- Optimize draw calls
- Use requestAnimationFrame for animations
- Clear canvas properly each frame
- Batch similar drawing operations
- Keep 60 FPS target

## Game Design
- Balance realism with simplicity
- Make UI kid-friendly (large buttons, emojis)
- Provide clear feedback for actions
- Use colors to indicate status (green=good, red=bad)
- Keep game mechanics intuitive
